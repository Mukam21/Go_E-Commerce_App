 ## Полнофункциональный онлайн-маркетплейс: покупатели и продавцы, каталог товаров, корзина, заказы и интеграция платежей — фронтенд на React и бэкенд на Go. Готов как для демонстрации навыков, так и для дальнейшего улучшения под реальный бизнес.

  # Основные функциональности:

- Регистрация / вход пользователей (JWT), роли buyer / seller.

- Верификация пользователя (SMS-код, dev-режим — лог вместо реальной отправки).

- Управление профилем пользователя (получение/обновление).

- Каталог товаров и категории (категорийный слайдер, карточки товаров, детальная страница товара).

- CRUD для продавцов: добавление/редактирование/удаление своих товаров, просмотр заказов продавца.

- Корзина: добавление/обновление/удаление товаров, подсчет стоимости.

- Оформление заказа + генерация заказа (order ref, items).

- Интеграция со Stripe — создание PaymentIntent, получение clientSecret (test mode).

- Хранение платежей в БД, проверка/верификация платежа.

- Загрузка файлов/изображений (FileUpload).

- Поиск и UI-компоненты: поиск, топ-товары, карусели/слайдеры, инпуты и кнопки (MUI).

- State management на фронтенде (Redux Toolkit + hooks), адаптация к TypeScript.

- Разделение кода: handlers / services / repositories (чистая архитектура).

  # Какие реальные проблемы решает проект:

- Онлайн-продажи: платформа позволяет продавцам выставлять товары, а покупателям — удобно находить и покупать.

- Разделение обязанностей: продавцы и покупатели работают в одной системе с разграничением прав.

- Безопасность и контроль: JWT-аутентификация + SMS-верификация повышают доверие и безопасность.

- Надёжные платежи: интеграция со Stripe (тестовый режим) для безопасной обработки карт.

- Управление заказами: автоматизация процесса — от корзины до создания заказа и очистки корзины.

- Масштабируемость и поддерживаемость: слоистая архитектура упрощает развитие функционала и тестирование.

- UX для пользователя: поиск, фильтрация, топ-товары, адаптивные элементы интерфейса — повышают конверсию.
